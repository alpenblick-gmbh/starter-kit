## Wann diese Docs nutzen?

1. **Vor der Implementierung** - Pruefe ob ein Guide existiert
2. **Waehrend `/plan`** - Fuer Architektur-Entscheidungen
3. **Waehrend `/dev`** - Fuer Code-Patterns

Dann hier in der Tabelle verlinken.
